# 🧠 Personal AI Engineering Training Program – 10 Weeks

## Inspired by the [Gauntlet AI](https://gauntletai.com/) Curriculum

> **Philosophy:** You don’t just *use* AI – you *build with* AI as your co-engineer. Every week escalates in complexity. Spec-driven development. Ship real systems. By Week 10, you’re building things unimaginable at Week 1.

**Two Phases. Escalating Complexity.**

| Phase | Weeks | Format | Focus |
|-------|-------|--------|-------|
| 🏁 | **Phase 1** | 1–3 | Remote / Solo | AI-first foundations, RAG, agents |
| ⛳ | **Phase 2** | 4–10 | Intensive | Delivery, enterprise systems, capstone |

**Tech Stack:** Node.js/TypeScript (Primary), Python (ML/Fine-tuning)
**IDE:** Cursor

---

## 🚩 Phase 1: Foundation (Remote – Weeks 1–3)
### Week 1 – AI-First Coding Methodologies
- AI-focused development workflows
- Coding with intelligent agents: Cursor, Claude Code, Codex
- Model Context Protocol (MCP)
- Real-time collaboration
- Spec-driven development
- **Mission:** Build a Node.js CLI based on a Markdown spec & MCP integration.

### Week 2 – Legacy Systems & RAG Implementation
- Retrieval-Augmented Generation (RAG)
- Vector DBs (Pinecone/Weaviate/Qdrant)
- Embeddings, retrieval, chunking
- Modernize legacy Python code via Node.js RAG chatbot
- **Mission:** Use Node.js to create a Q&A bot over an old Python codebase.

### Week 3 – Open Source Domain-Specific AI Agents
- Agent frameworks & protocols
- Observability (LangSmith/Helicone)
- Agent evals & systematic dev
- **Mission:** Ship a TypeScript "PR Reviewer Agent" with automated evals.

---

## 🚩 Phase 2: Application, Scale & Complexity (Weeks 4–10)
### Week 4 – Simulated Client Project I: Delivery
- Translate ambiguous requirements, professional deployment
- Use AI for scoping/spec/code/testing
- **Mission:** Ship a SaaS feature under time constraints.

### Week 5 – Simulated Client Project II: Speed & QA
- Rapid refactoring & feature integration
- AI-assisted test authoring (Jest/Mocha)
- **Mission:** Fast-follow changes to prior codebase with high test coverage.

### Week 6 – Enterprise Systems & Fine-Tuning
- Supervised fine-tuning (Python: PEFT/LoRA/QLoRA)
- Large-scale systems, enterprise architecture
- **Mission:** Fine-tune an open source model, wrap as scalable Node.js API.

### Week 7 – Legacy Modernization & Multi-Agent Systems
- Multi-agent system patterns, modernization pipelines
- **Mission:** Agents that coordinate to modernize a legacy repo (Planner, Coder, Reviewer, QA).

### Week 8 – Multimodal AI & Game Development
- Text, image, audio, and interactive system integration
- **Mission:** Build an AI-powered web app/game using vision/audio generation.

### Week 9 – Capstone & AI Infrastructure
- Large-scale agent/MCP orchestration
- Cloud infra (AWS/GCP), monitoring
- **Mission:** Start design/build for a scalable AI product.

### Week 10 – Capstone Completion & RL
- Reinforcement learning integration, final deployment
- Performance optimization, learning loops
- **Mission:** Complete, deploy, present “demo day” for a production AI system.

## How to Use This Repo
1. Each week, follow the `assignment/<week-X>.md` spec (see Issues).
2. Use templates in `/templates/` for each project spec, eval suite, and progress log.
3. Track daily progress in `/logs/`. Log blockers as issues.
4. Copy `/weekly-reflection-template.md` for your end-of-week retros.
5. Push code daily. Your GitHub history = proof of progress.
